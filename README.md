# 🕵 Доска расследования

Интерактивная доска связей в детективном стиле. Это один файл — `index.html`. Внутри сразу всё: разметка, оформление и данные. Ничего собирать или устанавливать не нужно — файл открывается двойным кликом в любом браузере и так же выкладывается в интернет.

---

## Что лежит в проекте

- **index.html** — сама доска. Весь контент (карточки людей, проекты, связи между ними, таймлайн) находится прямо внутри этого файла.
- **README.md** — этот файл с инструкцией.
- **.gitignore** — служебный файл, можно не трогать.

---

## Как редактировать содержимое

Все данные доски лежат в начале файла `index.html`, в четырёх списках:

- **N** — карточки: люди и проекты со всеми их полями и фактами.
- **E** — связи (линии) между карточками.
- **ZONES** — рамки-группы вокруг карточек (например, «Academix Hub»).
- **TL** — лента событий внизу доски.

Чтобы что-то поменять, правится только содержимое этих списков. Само оформление трогать не обязательно.

Проще всего просить Claude: открыть проект и сказать словами, что добавить или поправить — например «добавь нового человека, свяжи его с Игорем, достоверность средняя, вот источник». Claude сам внесёт правки в нужное место.

**Одно важное правило проекта:** ни один факт не попадает на доску без источника. Если подтвердить нечем — отмечаем его как «не подтверждено», а не выдаём за факт.

---

## Как обновлять доску (повседневная работа)

Когда правки готовы, их нужно отправить на GitHub — оттуда сайт обновится сам.

Открой командную строку в папке проекта и выполни три команды **по очереди** (каждую отдельно, с Enter):

```
git add .
git commit -m "что изменил"
git push
```

Через минуту-другую сайт обновится автоматически.

> ⚠️ **Если вы работаете вдвоём** — перед началом правок всегда сначала забирай свежую версию командой:
> ```
> git pull
> ```
> Это нужно, потому что вы оба меняете один и тот же файл. Если пропустить этот шаг и вы оба поправите файл одновременно, при отправке возникнет конфликт. Тогда просто покажи Claude текст ошибки — он подскажет, что нажать.

## Как выложить сайт в интернет через GitHub Pages

Это делается **один раз**. После настройки сайт будет сам обновляться при каждой отправке правок.

1. Открой свой репозиторий на сайте GitHub: `https://github.com/stylist111/detective-board`
2. Сверху нажми вкладку **Settings** (Настройки).
3. В меню слева выбери **Pages**.
4. В разделе **Build and deployment**, в поле **Source**, выбери **Deploy from a branch** (развёртывать из ветки).
5. Ниже появятся два выпадающих списка:
   - Первый (ветка): выбери **main**.
   - Второй (папка): выбери **/ (root)** — то есть корень.
6. Нажми **Save** (Сохранить).

Подожди 1–2 минуты и обнови страницу настроек. Сверху появится адрес вида:

```
https://stylist111.github.io/detective-board/
```

Это и есть твой живой сайт. Можешь сразу открыть и проверить.

> Первый раз сайт может появиться не мгновенно — иногда нужно подождать пару минут. Если открывается пустая или старая страница, подожди ещё немного и обнови (иногда помогает обновление с очисткой кэша: Ctrl + F5).

## Как проверить доску у себя, не выкладывая в интернет

Просто найди файл `index.html` в папке и открой двойным кликом — он откроется в браузере. Так удобно смотреть правки перед отправкой.

---

## Если что-то пошло не так

- **Команды не срабатывают / ошибки в консоли** — вводи команды строго по одной, каждую завершай нажатием Enter, не склеивай в одну строку.
- **При отправке пишет про конфликт или «rejected»** — скорее всего, на GitHub есть изменения, которых нет у тебя. Сделай `git pull`, затем снова `git push`. Если появятся непонятные сообщения — покажи их Claude.
- **Сайт не обновился** — подожди пару минут (GitHub Pages обновляется не моментально) и обнови страницу через Ctrl + F5.
